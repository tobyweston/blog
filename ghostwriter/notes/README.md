# Notes

Rough ideas, fragments, and observations.

These are **author thinking notes**, not finished writing.

Examples:

- possible post angles
- arguments
- story fragments
- examples from real work
- rough outlines

## Structure

Free-form Markdown.

Example:

```
code-review-thoughts.md
delivery-friction-observations.md
```

No strict format required.

## Usage

Notes can be supplied during planning:

```shell
python plan_post.py \
    --topic "Code review trust problem" \
    --notes-file notes/code-review-thoughts.md
```

## Rules

- Short and messy is fine
- Multiple unrelated ideas are OK
- Think of this as a scratchpad for the planner
