topics:

when_to_use:
- when the topic is pull requests, code review, branch protection, or review evidence
- when discussing trust and change assurance

summary: >
Code review is not only a collaboration technique. In many environments it also acts as
an evidence system for change quality, accountability, and delivery trust.

key_claims:
- Review practices shape both engineering behaviour and governance evidence.
- Tooling decisions can distort review quality while preserving only the appearance of control.
- Review systems should optimise for both feedback quality and trustworthy evidence.

argument_patterns:
- begin with code review practice or tooling
- show the tension between collaboration and control
- explain how review becomes evidence in regulated settings
- discuss side effects on trust, flow, and team behaviour

example_phrasing:
- Code review is doing more than most teams think.
- A control that preserves paperwork but damages trust is not much of a control.
- The review mechanism shapes the behaviour it is meant to observe.

guardrails:
- avoid treating pull requests as universally good or bad
- focus on trade-offs and context
