Running JUnit tests in parallel

Dec 29th, 2009

I’ve been playing with running tests in their own threads for a while now (in particular with reference to GUI testing) and am settling on a warm fuzzy feeling towards my general approach. I’m trying to capture this warm feeling more explicitly in the tempus-fugit project and today I was working on running tests in parallel with JUnit.

Looking more into recent versions of JUnit, there seems to be lots and lots of integration points for you to play with. I’ve been playing with Rules, Statements and Runners mostly and when creating your own BlockJUnit4ClassRunner, I spotted you can override the scheduler which, er, schedules the actual test methods to be run.
